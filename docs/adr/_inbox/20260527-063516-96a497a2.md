## ADR Bun.Image vs sharp — методологічно коректний бенчмарк і уточнена рекомендація

**Контекст:** Попередній ADR (`20260526-054228-bun-image-benchmark.md`) виніс вердикт «залишити sharp» на основі замірів із помилковою методологією: Bun.Image тестувалась без `palette: true` для PNG і без `progressive`/`mozjpeg` для JPEG, тобто не в apples-to-apples умовах. Виникло питання про достовірність того висновку.

**Рішення/Процедура/Факт:**
Побудовано окремий harness у `bench/` (не змінює `npm/src/`):
- `bench/download-corpus.mjs` — завантажує Kodak Lossless True Color Image Suite (24 PNG, 768×512) із `r0k.us/graphics/kodak/kodak/` (шлях подвоєний, не очевидно)
- `bench/codecs/sharp.mjs` та `bench/codecs/bun-image.mjs` — адаптери з однаковою сигнатурою `encode(buf, fmt)`; параметри зеркалять CLI: `palette:true`, `compressionLevel:9`, `progressive:true`, `mozjpeg:true`; для Bun.Image зафіксовано що `mozjpeg` і `effort` silently ігноруються
- `bench/quality.mjs` — SSIM через `ssim.js` (pure JS) + DSSIM через `dssim` CLI (Kornelski, brew); декодування назад у RGBA через sharp (нейтральний декодер)
- `bench/micro.mjs` — N=10 прогонів, відкидається перший (JIT warmup), median+p95; підтримує `--corpus` для довільного корпусу
- `bench/e2e.mjs` — 3 full-CLI прогони, медіана wall-clock; кеш видаляється між прогонами
- `bench/report.mjs` → `docs/bench/2026-05-26-bun-image-vs-sharp.md`
- Нове ADR: `docs/adr/_inbox/20260526-224541-bun-image-revisited.md`

Проведено два заміри:
1. **Kodak (24 photo-PNG):** Bun.Image −20% PNG, паритет AVIF, 8.94× швидший e2e, −19% output
2. **UI-корпус (25 PNG з lumik-ru/bono, 0.25–1.21 MB):** Bun.Image −38% PNG, AVIF **+38%** (програш), JPEG SSIM +0.133

**Обґрунтування:** Корпус Kodak — академічний стандарт image-codec бенчмарків, репродукований. SSIM+DSSIM необхідні для lossy-форматів: менший файл при гіршій якості не є перемогою. Parameter parity критична — різниця в `palette` дала 3× різницю у розмірі PNG (32 KB → 10 KB на ready.png). UI-корпус додано тому, що основний use-case проєкту — UI-активи та Tauri-іконки, а не фотографії.

**Розглянуті альтернативи:**
- Повне видалення sharp: відхилено — GIF encoder відсутній у Bun.Image, Linux backend невідомий, AVIF на UI програє
- Тільки мікробенч без e2e: відхилено — e2e показав що Bun.Image init-overhead відсутній (тобто мікрорезультати масштабуються)
- butteraugli замість DSSIM: відхилено — складніший в інтерпретації, SSIM+DSSIM достатньо

**Зачіпає:**
- `bench/` (новий каталог, 11 файлів harness) — не в workspace root
- `docs/bench/2026-05-26-bun-image-vs-sharp.md` — звіт із таблицями і висновком
- `docs/adr/_inbox/20260526-224541-bun-image-revisited.md` — оновлений ADR
- Рекомендація: гібрид — PNG/JPEG/WebP → Bun.Image, AVIF/GIF → sharp; `npm/src/index.js` НЕ змінювався (потрібен спочатку Linux Docker replication)
