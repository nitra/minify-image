## ADR Replace sharp → Bun.Image у @nitra/minify-image 4.0.0 (видалення GIF, NODE_ENV runCli fix)

**Контекст:** Benchmark-сесія на Kodak (photo PNG) та реальному UI-корпусі (lumik-ru/bono, 25 файлів 0.25–1.21 MB) показала, що `Bun.Image` (macOS, backend = Apple ImageIO/ANE) дає 4–9× прискорення при паритеті або перевазі по розміру для PNG/JPEG/AVIF на photo. Єдиний виняток — AVIF на flat UI-контенті (+38% розміру). Аналіз `~/www` виявив лише ~7 унікальних великих GIF (Bono-анімації), що унеможливлює обґрунтування підтримки всього `sharp` заради них.

**Рішення/Процедура/Факт:**
- `npm/src/index.js` — `import sharp` видалено; compressors `.jpeg`/`.jpg`/`.png` переписано через `new Bun.Image(buf).<format>().bytes()`; `writeAvif` переведено на `Bun.Image`; `.gif` більше не в glob; у `processOne` доданий `consola.warn('GIF compression removed in 4.0, file skipped: ...')` + ранній return.
- `npm/package.json` — версія `3.6.0 → 4.0.0`, `sharp` видалено з `dependencies`.
- `npm/CHANGELOG.md` — запис `[4.0.0]` з секціями `### Changed` (кодек) і `### Removed` (sharp, GIF), фактична метрика −17 MB footprint (macOS arm64, з 28 MB до 11 MB).
- `demo/test/run.test.js` — `'minified.gif'` прибрано з `expectedFiles`; додано assertions на GIF warn у stderr та відсутність GIF у TSV cache; `runCli` отримав `NODE_ENV: 'production'` в env дочірнього процесу (обхід Bun 1.3.14 bug: `bun:test` встановлює `NODE_ENV=test`, що глушить `consola.info` у spawned процесах → stdout порожній у тестах).
- `.jscpd.json` — додано `docs/superpowers/**` до `ignore` (spec і план повторювали один і той самий код-блок compressors).

**Обґрунтування:** AVIF на UI-корпусі — єдиний кейс, де Bun.Image програє (розмір +38%), але UI-AVIF у реальних проектах дає лише ~100–500 KB сумарно — прийнятна ціна за виключення native binary `sharp` (~17 MB залежностей). GIF — 87% файлів < 10 KB (loaders/spinners), усі великі (7 унікальних) належать до Bono-анімацій що рідко змінюються — утримувати 17 MB native build заради них нераціонально.

**Розглянуті альтернативи:**
- **(B) sharp залишається тільки для AVIF** — відхилено: залежність лишалась, net-win мінімальний.
- **(C) прибрати AVIF разом із GIF** — відхилено: AVIF на photo-корпусі паритет із 9× speedup — цінний формат.
- **Для GIF-поведінки: silent skip** — відхилено: хтось у CI вважатиме, що GIF стискаються, а їх ні; **hard exit** — відхилено: ламає build без потреби.

**Зачіпає:** `npm/src/index.js`, `npm/package.json`, `npm/CHANGELOG.md`, `demo/test/run.test.js`, `.jscpd.json`; публічний API `@nitra/minify-image` (breaking: GIF видалено, мін. Bun ≥ 1.3 вже вимагалось).
