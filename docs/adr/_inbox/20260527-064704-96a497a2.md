## File Structure

Before defining tasks, map out which files will be created or modified and what each one is responsible for. This is where decomposition decisions get locked in.

- Design units with clear boundaries and well-defined interfaces. Each file should have one clear responsibility.
- You reason best about code you can hold in context at once, and your edits are more reliable when files are focused. Prefer smaller, focused files over large ones that do too much.
- Files that change together should live together. Split by responsibility, not by technical layer.

## ADR Заміна sharp на Bun.Image у `@nitra/minify-image` (v4.0)

**Контекст:** У версії 3.6.0 `@nitra/minify-image` використовував `sharp` як єдиний бекенд стискання растрових зображень (PNG, JPEG, GIF) і генерації AVIF. `sharp` тягне ~16.6 MB native-бінарок (`@img` prebuild). Після методологічно коректного бенчмарку (Kodak 24 PNG + реальний UI-корпус, SSIM/DSSIM, apples-to-apples параметри) виявлено, що `Bun.Image` (вбудований у Bun 1.3.14+ API, backend `system` = Apple ImageIO) є конкурентоспроможною заміною.

**Рішення/Процедура/Факт:**
- `npm/src/index.js`: замінено `.png`, `.jpeg/.jpg`, `.webp`, `.avif` compressors з `sharp` → `Bun.Image`
- `npm/src/index.js`: `.gif` compressor видалено; при виявленні `.gif` файлу CLI виводить `consola.warn` і пропускає файл
- `npm/package.json`: `sharp` видалено з `dependencies`; додано `engines.bun >= 1.3.14`
- `npm/CHANGELOG.md`: задокументовано як breaking change v4.0.0
- `globSync` pattern: `**/*.gif` залишено у glob (щоб warn-повідомлення спрацьовувало), але compressor для `.gif` — відсутній

**Обґрунтування:**
- Micro-bench (Kodak 24 photo-PNG): Bun.Image 1.95–9.23× швидше залежно від формату; розмір output — паритет або краще для PNG (−20%), паритет для AVIF (+0.3%), кращий SSIM на UI для JPEG.
- E2E bench (Kodak + demo files): Bun.Image 8.94× швидше, output −19%.
- GIF: 87% GIF-файлів у корпусі < 10 KB (loaders); 7 унікальних великих GIF — стиснуті одноразово; `Bun.Image` не має GIF encoder.
- На macOS використовує Apple ImageIO + ANE/Media Engine HW-acceleration, що пояснює speedup. На Linux backend — libvips або інший (потребує верифікації).
- Видалення `sharp` як dep: −16.6 MB native binaries при `npm install`.

**Розглянуті альтернативи:**
- **Гібрид A (sharp для GIF + AVIF):** залишити sharp лише для форматів де Bun.Image програє (AVIF UI +38%). Відхилено — складніший dep-tree без суттєвої вигоди для реального корпусу.
- **Гібрид B (sharp для AVIF, Bun.Image для PNG/JPEG):** компроміс. Відхилено — `sharp` лишається як dep, але AVIF на UI-корпусі: +38% більший файл — прийнятна ціна за повне видалення native dep.
- **Видалення AVIF:** прибрати `--avif` прапор. Відхилено — AVIF на photo-корпусі (паритет розміру + 9.23× швидше) — реальний win.

**Зачіпає:** `npm/src/index.js`, `npm/package.json`, `npm/CHANGELOG.md`, публічний API (breaking: GIF видалено), `demo/test/run.test.js`, `demo/test/avif-opt-out.test.js`
