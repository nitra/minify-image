## ADR Per-package opt-out для генерації AVIF через `package.json`

**Контекст:** `@nitra/minify-image` з прапорцем `--avif` створював `.avif`-двійники для всіх растрових зображень, включно з пакетами монорепи, що не підтримують AVIF (наприклад, Capacitor-проєкти). `@nitra/cursor` вже вмів читати opt-out з `package.json`, але мініфаєр його ігнорував — дві сторони розходилися щодо того, які файли мають існувати.

**Рішення/Процедура/Факт:**

- У `npm/src/index.js` додано три допоміжних об'єкти: `dirAvifOptOutCache` (`Map<string, boolean>`, ключ — абсолютний шлях каталогу), `readDirAvifOptOut` (читає найближчий `package.json` і повертає tri-state: `true` / `false` / `null` якщо файл відсутній або пошкоджений), `cacheAvifOptOut` (записує результат у всі відвідані каталоги за один прохід), `isAvifOptedOut` (walk-up від каталогу зображення до `srcAbs` — зупиняється на першому знайденому `package.json`).
- Семантика «зупинись на першому `package.json`»: opt-out прив'язаний до найближчого workspace-пакета; якщо він не містить прапорця — далі вгору не шукати (інакше root-`package.json` без прапорця не мав би ефекту, що очікуваний).
- Умова обчислення `avifPath` у `processOne` доповнена `&& !isAvifOptedOut(imagePath)`. Звичайне стиснення оригіналу **не змінюється**.
- Формат конфігурації: `{ "@nitra/minify-image": { "disable-avif": true } }` у `package.json` workspace-пакета.
- Версія піднята до `3.4.0`; оновлено `CHANGELOG.md`, `README.md`, `--help`.
- Додано `demo/test/avif-opt-out.test.js` з 5 тестами: генерація поза/всередині opt-out-пакета, стиснення оригіналів у opt-out-пакеті, наявні `.avif` не чіпаються, пошкоджений `package.json` не падає, `"disable-avif": false` → AVIF створюється.

**Обґрунтування:** Opt-out — це властивість пакета (його ставить мейнтейнер у репо), а не контексту запуску. CLI-флаг типу `--no-avif-for=<glob>` створив би розсинхрон між cursor і мініфаєром. Єдина точка істини — `package.json` workspace-пакета; саме цей формат вже читає `@nitra/cursor` image-avif. Кеш по каталогу гарантує, що при тисячах зображень у пакеті `package.json` читається рівно один раз. Видалення вже наявних `.avif` в opt-out-пакетах — відповідальність cursor cleanup (або разової команди `find <pkg> -name "*.avif" -delete`), не мінімайзера.

**Розглянуті альтернативи:** CLI-флаг `--no-avif-for=<glob>` — відхилено через розсинхрон із cursor. Глобальний конфіг-файл — не обговорювався. Модифікація `AVIF_IGNORE_PATH_RE` — не підходить: path-based skip — окрема конкретна семантика, не те саме, що per-package opt-out.

**Зачіпає:** `npm/src/index.js` (helpers `dirAvifOptOutCache`, `readDirAvifOptOut`, `cacheAvifOptOut`, `isAvifOptedOut`; умова `avifPath` у `processOne`; `--help`), `npm/package.json` (версія `3.4.0`), `npm/CHANGELOG.md`, `npm/README.md`, `demo/test/avif-opt-out.test.js` (новий файл).
