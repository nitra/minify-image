## ADR Оновлення `svgo` 3.3.2 → 4.0.1 і нові дефолти `preset-default`

**Контекст:** Рутинний `/n-taze` підняв `svgo` на major-версію (3→4); єдина точка дотику — `npm/src/index.js:323`, виклик `optimize(stripped, { plugins: [{ name: 'preset-default', params: { overrides: {...} } }] })`.

**Рішення/Процедура/Факт:** Оновлення прийнято без змін у коді. У `svgo` v4 `preset-default` змінено два дефолти: `removeViewBox` і `removeTitle` тепер **вимкнені** (раніше вмикались). Проєкт не перевизначав ці плагіни в `overrides`, тож автоматично переходить на безпечніший дефолт — `viewBox` і `<title>` зберігаються у вихідних SVG. Всі 6 SVG-тестів залишились зеленими.

**Обґрунтування:** Новий дефолт краще з точки зору масштабованості (SVG без `viewBox` не масштабується через CSS) і доступності (`<title>` — screen-reader текст). Якщо для конкретного проєкту важливий мінімальний розмір — `removeViewBox`/`removeTitle` можна явно додати до `overrides` у `npm/src/index.js:324`.

**Розглянуті альтернативи:** Явне перевизначення `removeViewBox: true` і `removeTitle: true` у `overrides` для збереження старої поведінки — відхилено, бо безпечніший дефолт переважає.

**Зачіпає:** `npm/src/index.js` (виклик `svgoOptimize`), `npm/package.json` (залежність `svgo`), `npm/CHANGELOG.md`.

## ADR `**/src-tauri/icons/**` у always-on exclude-списку + AVIF-скіп

**Контекст:** Tauri-проєкти зберігають PNG-іконки у `src-tauri/icons/` (генерується через `tauri icon` CLI). Квантизація RGBA-PNG у palette (color type 6 → 3) ламає `tauri::generate_context!` з panic «icon … is not RGBA». До цього `minify-image` обробляв цю директорію, якщо не вказати `--ignore` вручну.

**Рішення/Процедура/Факт:** `**/src-tauri/icons/**` додано до always-on `globOptions.ignore` поряд з `node_modules`, `vendor`, `test`, `dist`, `**/.*/**` (`npm/src/index.js` ~рядок 83). Та сама директорія додана до `AVIF_IGNORE_PATH_RE` як друга лінія оборони (AVIF поряд з `.icns`/`.ico` безглуздий). Написано два regression-тести у `demo/test/tauri-icons-default-ignore.test.js`: файл під `src-tauri/icons/` не змінюється без додаткових прапорців; `.avif` поряд із ним не створюється навіть з `--avif`. Для матчу глобів використано `path.matchesGlob` (Node.js built-in, Bun підтримує) замість рантайм-залежності `picomatch`. Оновлено `--help`, README (розділ «Tauri / icon assets»), CHANGELOG (3.6.0).

**Обґрунтування:** Канонічний шлях Tauri фіксований стандартом CLI — ймовірність false-positive мінімальна. Default-exclude дає нульову регресію для Tauri-проєктів без будь-яких прапорців. `path.matchesGlob` (стандарт Node ≥ 22.5) дозволяє уникнути нової рантайм-залежності.

**Розглянуті альтернативи:** Явний `--preserve-color-type=<glob>` CLI-прапорець і per-package config у `package.json` — запропоновано у специфікації, але явно відхилено під час реалізації; реалізована лише перша лінія (default-ignore). Розширення exclude на `app-icons/`, `electron/build/icons/` тощо — відкладено як окремий issue.

**Зачіпає:** `npm/src/index.js` (`globOptions.ignore`, `AVIF_IGNORE_PATH_RE`), `npm/README.md`, `npm/CHANGELOG.md`, `demo/test/tauri-icons-default-ignore.test.js`, `.cspell.json` (додано слово `tauri`).

## Knowledge Відхилений `--preserve-color-type` API під час сесії

**Контекст:** Специфікація вимагала `--preserve-color-type=<glob>` CLI-прапорця (repeatable) і аналогічного поля в `package.json` споживача, що повторює шаблон `disable-avif`. Функція мала пропускати квантизацію palette/bit-depth для PNG-файлів, що матчать glob, але залишати lossless-оптимізації.

**Рішення/Процедура/Факт:** Під час реалізації (Task 4) функцію explicitly скоуп-аутовано. Код `parseArgs`, резолвер і гілка PNG-компресора, що вже були додані, — повністю відкочені. Реалізований тільки defensive default (`src-tauri/icons/**` у ignore).

**Обґрунтування:** Не зафіксоване явно; вирішено в ході сесії. Для Tauri-кейсу достатньо default-ignore — файли просто не обробляються, і color-type збережено без будь-якого API.

**Розглянуті альтернативи:** Повна реалізація per-CLI-flag + per-package config — відхилена в рамках цього патчу.

**Зачіпає:** `npm/src/index.js` (parseArgs, options, compressors), `npm/package.json` (залежність `picomatch` додана та видалена).
