## ADR Методологічно коректний бенчмарк Bun.Image vs sharp (Kodak + SSIM/DSSIM + palette parity)

**Контекст:** Попередній ADR (`20260526-054228-bun-image-benchmark.md`) тестував Bun.Image без `palette: true` для PNG та без `mozjpeg`/`progressive` для JPEG — тобто порівнював різні конфігурації, а не однакові умови. Висновок «sharp значно кращий» міг бути артефактом методологічної помилки.

**Рішення/Процедура/Факт:** Впроваджено harness `bench/` із адаптерами (`bench/codecs/sharp.mjs`, `bench/codecs/bun-image.mjs`) з однаковими параметрами (PNG: `compressionLevel:9, palette:true`; JPEG: `progressive:true, quality:75`; AVIF: `quality:40`). Корпус — Kodak Lossless True Color Image Suite (24 PNG, 768×512). Метрики: розмір, SSIM (`ssim.js`), DSSIM (`dssim` CLI). Мікро-бенч: 10 прогонів, медіана+p95. E2E: 3 прогони CLI, медіана wall-clock.

**Обґрунтування:** Без apples-to-apples параметрів і стандартного corpus (Kodak — академічний стандарт) висновки не відтворювані. `mozjpeg` і `effort` у Bun.Image silently ignored — зафіксовано у звіті явно.

**Розглянуті альтернативи:** Власний корпус `~/www/nitra` (нерепродуковано), лише розмір без SSIM/DSSIM (не виявляє quality trade-off), butteraugli (складніше, зайве).

**Зачіпає:** `bench/`, `docs/bench/2026-05-26-bun-image-vs-sharp.md`, `docs/bench/2026-05-27-kodak-png-to-avif.md`

## Knowledge Bun.Image AVIF: photo vs UI-контент дають протилежні результати

**Контекст:** Після методологічно коректного бенчмарку виникло питання чи можна повністю замінити sharp на Bun.Image, включно з AVIF.

**Рішення/Процедура/Факт:** На Kodak photo corpus (768×512 фото): Bun.Image AVIF ≈ паритет розміру з sharp (+0.3%), але 9.23× швидший. На UI-корпусі (`lumik-ru/static/bg/` + `bono/loyalty/assets/`, 25 PNG 0.25–1.21 MB): Bun.Image AVIF дає **+38% більший файл** ніж sharp. PNG на UI-корпусі навпаки: Bun.Image −38% менший (4.33× швидший). JPEG на UI: Bun.Image +12% розміру але SSIM +0.133 (кардинально якісніший).

**Обґрунтування:** `Bun.Image.backend = system` на macOS — Apple ImageIO + ANE/Media Engine. ImageIO AVIF encoder оптимізований під photo-content (градієнти, текстури), але слабший на flat UI fills. PNG encoder через palette quantization виграє на UI більше ніж на photo.

**Розглянуті альтернативи:** Не обговорювалися — факт виявлений емпірично.

**Зачіпає:** Рішення про AVIF в `@nitra/minify-image`, `docs/bench/2026-05-26-bun-image-vs-sharp.md` (секція UI corpus)

## ADR Replace sharp → Bun.Image у @nitra/minify-image 4.0.0, видалення GIF

**Контекст:** sharp (~16.6 MB native binaries: `sharp` 596 KB + `@img` 16 MB) — єдина нативна залежність пакету. Bun.Image (Bun ≥1.3, вже обов'язковий) покриває PNG/JPEG/AVIF/WebP без додаткових deps. Аналіз GIF у `~/www` показав 196 файлів із яких 87% (<10 KB loaders/spinners) і лише ~7 унікальних великих анімацій (Bono brand, 1.5–4 MB), які рідко змінюються.

**Рішення/Процедура/Факт:** `npm/src/index.js` версії 4.0.0: `import sharp` видалено, PNG/JPEG/AVIF переведені на `Bun.Image`. `.gif` залишений у `compressors` map але повертає `null` + `consola.warn('GIF compression removed in 4.0, file skipped: <path>')`. `AVIF_SOURCE_EXTS` — `.gif` видалено. `sharp.cache(false)` / `sharp.concurrency(1)` — видалено. `sharp` видалено з `npm/package.json` dependencies. HELP_TEXT оновлено (GIF прибрано). Версія `3.6.0 → 4.0.0` (major bump = breaking).

**Обґрунтування:** E2E результат: Bun.Image 8.94× швидший за sharp, output −19% менший на Kodak corpus. Нетрь sharp = −16.6 MB на macOS arm64 (на Linux ще більше через мультиплатформні binaries). GIF: 87% файлів < 10 KB (overhead > економія), 7 унікальних великих анімацій — неекономічно тримати весь sharp заради них. AVIF на UI-контенті з Bun.Image +38% — прийнятна ціна для повного видалення sharp.

**Розглянуті альтернативи:** (A) Повний перехід на Bun.Image — **обрано**. (B) sharp лише для AVIF — dep лишається, менше виграш. (C) Видалити AVIF взагалі — breaking change без потреби. Для GIF: залишити sharp (відкинуто), gifsicle wasm (ще одна dep), міграція на WebP/APNG (майбутнє).

**Зачіпає:** `npm/src/index.js`, `npm/package.json` (version + dependencies), `npm/CHANGELOG.md`, `demo/test/run.test.js` (GIF-тести, очікувані файли), `docs/superpowers/specs/2026-05-27-replace-sharp-with-bun-image.md`, `docs/superpowers/plans/2026-05-27-replace-sharp-with-bun-image.md`
