## ADR Tauri icons — defensive default exclude + AVIF skip у `@nitra/minify-image`

**Контекст:** Пакет `@nitra/minify-image` застосовував palette-квантизацію до RGBA-PNG у `src-tauri/icons/**`, що призводило до падіння `tauri::generate_context!` (panic «icon … is not RGBA») у проєктах на Tauri. Те ж стосується AVIF-генерації — Tauri вбудовує ці файли безпосередньо, тому `.avif`-супутники там безглузді.

**Рішення/Процедура/Факт:**
- `**/src-tauri/icons/**` додано до always-on glob ignore у `tinyglobby`-конфігурації (`npm/src/index.js`, масив `globOptions.ignore`), поряд із `node_modules`, `vendor`, `test`, `dist`, `**/.*/**`. Файли під цим шляхом тепер пропускаються цілком — не торкаються ні PNG-компресором, ні AVIF-енкодером.
- `src-tauri/icons` додано до `AVIF_IGNORE_PATH_RE` (`npm/src/index.js`) — друга лінія оборони: навіть якщо глоб-ignore перекрити через `--ignore`-override, AVIF не генерується.
- HELP_TEXT для `--ignore` оновлено (список дефолтів).
- Написано 2 regression-тести у `demo/test/tauri-icons-default-ignore.test.js`: (a) RGBA-PNG 256×256 під `src-tauri/icons/` лишається байт-у-байт незміненим при `--write`; (b) `--avif` не генерує `.avif` для файлів під цим шляхом.
- Версія `npm/package.json` bump: `3.5.0 → 3.6.0`. `npm/CHANGELOG.md` і `npm/README.md` (нова секція «Tauri / icon assets») оновлено.

**Обґрунтування:** Канонічна структура Tauri (генерується `tauri icon` CLI) завжди кладе іконки у `src-tauri/icons/`. Шанс false-positive мінімальний — лише Tauri-проєкти мають цей шлях, і для них будь-яка оптимізація кольорового простору руйнівна. Defensive default (без додаткових прапорців) — правильний UX: не чіпати, якщо не попросили явно. Реалізація рівня «--preserve-color-type API» (per-file, per-package, CLI-прапорець) була свідомо відхилена замовником як надлишкова в рамках цього патчу.

**Розглянуті альтернативи:**
- `--preserve-color-type=<glob>` CLI-прапорець + per-package поле `{ "@nitra/minify-image": { "preserve-color-type": [...] } }` — заплановано у деталях (план у `docs/superpowers/plans/`), але відхилено після review: перший рівень (defensive default) покриває 99% випадків без API-складності.
- `picomatch` / `path.matchesGlob` для кастомного glob-матчингу — розглядалося для preserve-резолвера, виявилося непотрібним без Level 2 API; не додано.

**Зачіпає:**
- `npm/src/index.js` — `globOptions.ignore`, `AVIF_IGNORE_PATH_RE`, HELP_TEXT
- `demo/test/tauri-icons-default-ignore.test.js` — новий regression-тест
- `npm/CHANGELOG.md`, `npm/README.md`, `npm/package.json` (версія `3.6.0`)
- `.cspell.json` — слово `tauri` додано до словника

## ADR C4-діаграми архітектури `@nitra/minify-image` у `docs/ci4/`

**Контекст:** Правило `.cursor/rules/n-ci4.mdc` вимагає, щоб C4-модель проєкту жила у Markdown поряд із кодом і оновлювалась у тому ж PR, що й архітектурні зміни. Жодного C4-файлу в репозиторії не було.

**Рішення/Процедура/Факт:**
- Створено `docs/ci4/README.md` із трьома Mermaid C4-діаграмами (`C4Context`, `C4Container`, `C4Component`) в одному файлі.
- **Level 1 (System Context):** Developer/CI → CLI → filesystem (`.n-minify-image.tsv`, `mtime.tsv`, image files, consumer `package.json`).
- **Level 2 (Container):** Один CLI-процес (Node.js 24 / Bun 1.3, ESM) + чотири external storage.
- **Level 3 (Component):** 9 компонентів усередині `npm/src/index.js` — argparse, glob walker, processOne, raster compressor (sharp), SVG compressor (SVGO 4), AVIF encoder, AVIF opt-out resolver, hash cache I/O, mtime cache I/O; таблиця «component → test files».
- Посилання «Architecture / system context» додано до `npm/README.md`.
- У процесі виправлено синтаксис Mermaid: видалено екрановані лапки `\"`, `\\t`, кутові дужки `<…>`, glob-фігурні `{}` та одинарні лапки у description-полях (Mermaid C4-парсер їх не підтримує — рендер обривався). Перевірено render через `@mermaid-js/mermaid-cli` (3 SVG згенеровано успішно).

**Обґрунтування:** Один файл `docs/ci4/README.md` рендериться нативно на GitHub при переході у директорію і читабельний як plain text. Три рівні C4 у одному місці дають вичерпну картину без «портальної» окремої сторінки документації, що узгоджується з духом правила `n-ci4.mdc` — «поряд із кодом, а не для адміністраторів архітектурного порталу».

**Розглянуті альтернативи:** Окремі файли на кожен рівень (`context.md`, `container.md`, `component.md`) — відхилено: надлишкова навігація для проєкту з одним entry-point (`npm/src/index.js`).

**Зачіпає:**
- `docs/ci4/README.md` — новий файл (C4 context + container + component)
- `npm/README.md` — посилання на `docs/ci4/`
- `.cspell.json` — слово `tauri` (спільно з попереднім ADR)
