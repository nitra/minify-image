## ADR Bun.Image vs sharp — замір з виправленою методологією (Kodak + UI-корпус, SSIM/DSSIM, palette parity)

**Контекст:** Попередній ADR (`20260526-054228-bun-image-benchmark.md`) зробив висновок «sharp залишається» на підставі заміру без `palette: true` для Bun.Image PNG та без SSIM/DSSIM. Потрібна переоцінка з апples-to-apples параметрами та стандартним image-codec корпусом.

**Рішення/Процедура/Факт:** Реалізовано `bench/` harness: `download-corpus.mjs` (Kodak 24 PNG), `codecs/{sharp,bun-image,index}.mjs` (4 adapter-варіанти з parity-параметрами), `quality.mjs` (SSIM via `ssim.js` + DSSIM via `dssim` CLI), `micro.mjs` (N=10, median+p95), `e2e.mjs` (3 CLI-прогони sharp vs fork), `report.mjs`. Запущено на Kodak-корпусі та реальному UI-корпусі (lumik-ru + bono, 25 PNG 0.25–1.21 MB).

**Обґрунтування:** Попередній замір ігнорував `palette: true` (64→94% compression win на PNG) і `mozjpeg` silently-ignore у Bun.Image; без `dssim` AVIF-порівняння за розміром нечесне. Kodak — академічний photo-стандарт, UI-корпус — реальний use case.

**Розглянуті альтернативи:** butteraugli (відхилено — складніше, DSSIM+SSIM достатньо), тільки розмір без якості (відхилено — нечесне для lossy), тільки мікробенч без e2e (відхилено — не показує реальний user-perceived час).

**Зачіпає:** `bench/` (новий каталог), `docs/bench/2026-05-26-bun-image-vs-sharp.md`, `docs/adr/_inbox/20260526-224541-bun-image-revisited.md`

## Knowledge Bun.Image AVIF на photo vs UI — протилежна поведінка розміру

**Контекст:** Під час бенчмарку виявлено, що перевага Bun.Image (Apple ImageIO) для AVIF повністю залежить від типу контенту.

**Рішення/Процедура/Факт:** На photo-корпусі Kodak Bun.Image AVIF ≈ паритет за розміром із sharp; на UI-корпусі (flat gradients, logos, background assets) — **+38% більший файл** при тих самих `quality: 40`. На PNG навпаки: Bun.Image краще стискає UI ніж photo (−38% vs −20%).

**Обґрунтування:** Apple ImageIO AVIF (використовується в `backend: system` на macOS) оптимізований для photo-контенту; libavif (sharp) кращий для flat/palette UI-контенту.

**Розглянуті альтернативи:** не обговорювались.

**Зачіпає:** вирішення «PNG/JPEG/WebP → Bun.Image, AVIF/GIF → sharp залишається»; майбутня migration `npm/src/index.js`.
