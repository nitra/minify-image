## ADR Оновлення залежностей до останніх версій (`/n-taze`) — `@nitra/minify-image` v3.5.0

**Контекст:** Проєкт `@nitra/minify-image` (v3.4.0) потребував оновлення залежностей через накопичені patch- та minor-версії в workspace-пакетах `demo` і `npm`. Запуск `/n-taze` ініціював процес через `bunx taze -w -r latest`.

**Рішення/Процедура/Факт:**

- `bunx taze -w -r latest` оновив: `@nitra/eslint-config` `^3.8.0 → ^3.9.3`, `sharp` `^0.34.1 → ^0.34.5`, `@nitra/cursor` `^1.8.212 → ^1.8.221`, `p-limit` patch.
- Перевірено, що `svgo` вже знаходився на `^4.0.1` (major-стрибок з v3 відбувся раніше).
- Досліджено breaking changes `svgo` v3→v4 через `WebFetch` GitHub releases: `removeViewBox` і `removeTitle` тепер вимкнені за замовчуванням; відкинуто підтримку Node.js v14.
- Зафіксовано 3 падаючих тести (`--write: реально перезаписує файл коли економія >15%` та ін.) — відновлено старі версії та підтверджено, що ці тести падали й до оновлення (pre-existing failures). SVG-специфічні тести (6 шт.) пройшли успішно.
- Версія пакета підвищена `3.4.0 → 3.5.0`; додано запис у `npm/CHANGELOG.md`.
- Видалено тимчасові `.taze-bak`-файли; запущено `bun run lint` (виявлено cspell-проблеми лише в `docs/adr/_inbox/*.md`, не пов'язані з оновленням залежностей).

**Обґрунтування:** Оновлення — патчеві/мінорні, жодного несумісного API-змінення у використовуваному коді не виявлено. `svgo` v4 змінює лише дефолти плагінів, що не впливає на `npm/src/index.js`, який передає явну конфігурацію до `svgoOptimize`. Minor-версія `3.5.0` виправдана тим, що оновлення зачіпають транзитивні інструменти (eslint-config, cursor), а не лише patch-фікси.

**Розглянуті альтернативи:** Не обговорювалися; `taze` з `latest`-режимом є стандартною процедурою проєкту.

**Зачіпає:** `package.json`, `demo/package.json`, `npm/package.json`, `bun.lock`, `npm/CHANGELOG.md`; публічний API `@nitra/minify-image` не змінювався.
