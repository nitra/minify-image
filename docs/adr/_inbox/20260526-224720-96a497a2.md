## ADR Bun.Image vs sharp — переоцінка з виправленою методологією (Kodak, SSIM/DSSIM, palette parity)

**Контекст:** Попередній ADR (`20260526-054228-bun-image-benchmark.md`) відхилив Bun.Image на основі методологічно некоректного тесту: без `palette: true` для PNG, без apples-to-apples якості, на 23 не репродуковних файлах, без SSIM/DSSIM. Виникла потреба у чесному повторному порівнянні.

**Рішення/Процедура/Факт:** Реалізовано standalone harness у `bench/` (Kodak 24-PNG corpus, адаптери `sharp.mjs`/`bun-image.mjs` з ідентичними параметрами, SSIM via `ssim.js`, DSSIM via Kornelski's CLI, мікро-бенч N=10 медіана/p95, E2E via CLI fork N=3). Результати: на macOS arm64 Bun.Image 8.9× швидше E2E, −22% PNG, −7% AVIF, якість SSIM/DSSIM не гірша за sharp.

**Обґрунтування:** Kodak suite — академічний стандарт для image-codec порівнянь (24 репрезентативних фото 768×512, lossless PNG). Параметри адаптерів вирівняні по `npm/src/index.js` (palette, progressive, quality). SSIM+DSSIM — стандарт для оцінки lossy codec без суб'єктивного перегляду. Silent-ignore опцій (`mozjpeg`, `effort`) явно зафіксовано у звіті.

**Розглянуті альтернативи:** butteraugli (відкинуто — надмірна складність), власні активи з `~/www/nitra/` (відкинуто — нерепродуковно для сторонніх), butteraugli+DSSIM combo (SSIM+DSSIM обрано як достатньо і стандартно).
